CI Pipeline with GitHub Actions & Docker

Designed and implemented an automated Continuous Integration workflow triggered on push to the main branch. The pipeline builds a Docker image from a custom Dockerfile and securely publishes the artifact to Docker Hub.

Objective

Build a repeatable CI workflow that:

  • Triggers automatically on code push
  • Builds a containerized application
  • Publishes a deployable Docker artifact
  • Handles credentials securely

Pipeline Architecture

Local → GitHub Repository → GitHub Actions Runner (Ubuntu) → Docker Build → Docker Hub Registry

  • Workflow defined using YAML configuration
  • Trigger: push to main branch
  • Runner environment: ubuntu-latest
  • Repository secrets injected at runtime
  • Docker image tagged and pushed automatically

Technical Components

Version Control

  • Git CLI-based workflow
  • Branch tracking and remote synchronization
  • Commit-triggered automation

GitHub Actions Workflow

  • Checkout repository step
  • Docker Hub login using encrypted secrets
  • Docker build command execution
  • Automated image push to registry

Containerization

  • Dockerfile using nginx:alpine base image
  • Static HTML/CSS packaged into container
  • Port 80 exposure for runtime execution

Validation & Testing

  • Verified successful workflow execution in Actions tab
  • Confirmed image availability in Docker Hub
  • Pulled image locally using docker pull
  • Executed container with port mapping
  • Validated browser output via localhost

Security Design

  • No credentials hardcoded in workflow
  • Repository secrets used for Docker authentication
  • Secrets injected only at runtime

Outcome

Successfully implemented an automated CI pipeline producing containerized deployable artifacts. Established foundational DevOps workflow integrating Git, Docker, and secure CI automation.
