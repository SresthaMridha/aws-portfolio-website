CI/CD Pipeline with GitHub Actions and Docker

Pipeline GitHub Actions
Container Docker
Registry Docker Hub
Trigger Git Push Automation

This project demonstrates a continuous integration and deployment pipeline that automatically builds and publishes container images whenever changes are pushed to the repository. The pipeline uses GitHub Actions to automate Docker image creation and push the image to Docker Hub.

Objective

  • Automate application builds using CI/CD
  • Create Docker images from application source code
  • Push images automatically to Docker Hub
  • Establish a reproducible deployment workflow

Pipeline Architecture

Developer Push → GitHub Repository → GitHub Actions Workflow → Docker Build → Push to Docker Hub → Container Deployment

Every code change triggers a GitHub Actions workflow that builds a Docker image using the repository's Dockerfile and publishes the image to Docker Hub.

GitHub Actions Workflow
  • Pipeline triggered on push to the main branch
  • Workflow runs on GitHub-hosted runners
  • Docker image built automatically
  • Authentication to Docker Hub via repository secrets
  • Image pushed to container registry
Containerization
  • Application packaged inside Docker container
  • Nginx used as the web server
  • Static web content served through container
Security Configuration
  • Docker Hub credentials stored as GitHub Secrets
  • Secrets injected securely during pipeline execution
  • No credentials exposed in repository code
Deployment Validation
  • Docker image successfully pushed to Docker Hub
  • Image pulled locally from registry
  • Container launched using Docker runtime
  • Application verified through browser access

Outcome

The pipeline successfully automated the container build and publishing process. Each code push produces a reproducible container artifact ready for deployment, reducing manual steps and improving deployment consistency.
