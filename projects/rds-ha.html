RDS Multi-AZ High Availability Database Deployment

Database MySQL 8.4
Availability RDS Multi-AZ
Storage gp3 · 200GB · 3000 IOPS
Backup Automated + Snapshots

This project demonstrates the deployment of a production-style relational database using Amazon RDS with Multi-AZ high availability. The setup includes automated backups, encryption, monitoring, and snapshot management to ensure durability and operational resilience.

Objective

  • Deploy a highly available MySQL database
  • Implement synchronous standby replication
  • Enable automated backups with point-in-time recovery
  • Secure the database and protect against accidental deletion

Architecture Overview

The database architecture follows AWS Multi-AZ deployment:

Application Layer → RDS Primary Instance → Synchronous Replication → Standby Instance (Secondary AZ)

The primary instance processes application read/write operations while the standby replica continuously synchronizes data in another availability zone. If the primary instance fails, AWS automatically performs failover to the standby instance.

Database Configuration

  • Engine: MySQL Community
  • Version: 8.4.7
  • Instance class: db.m7g.large
  • vCPU: 2
  • RAM: 8 GB
  • Storage type: General Purpose SSD (gp3)
  • Allocated storage: 200 GiB
  • Provisioned IOPS: 3000
  • Storage throughput: 125 MiB/s
  • Storage autoscaling enabled

High Availability Design

  • Multi-AZ deployment enabled
  • Synchronous replication to standby instance
  • Automatic failover handled by AWS
  • Standby instance remains inaccessible to applications

This configuration ensures minimal downtime during infrastructure failures.

Backup and Recovery Strategy

  • Automated backups enabled
  • Backup retention period: 7 days
  • Point-in-Time Recovery supported
  • Backup window configured
  • Manual snapshots created for additional restore points

Manual snapshots provide an additional recovery mechanism beyond automated backups.

Security Configuration

  • Database encryption enabled using AWS KMS
  • Public access disabled
  • Deletion protection enabled
  • Database accessible only from application layer

Monitoring and Observability

  • Enhanced Monitoring enabled
  • Performance Insights enabled
  • Monitoring granularity set to 60 seconds
  • IAM monitoring role configured

Monitoring metrics provide visibility into database performance, resource utilization, and query activity.

Operational Testing

  • Verified Multi-AZ configuration status
  • Confirmed automated backups functioning
  • Created manual snapshot and verified availability
  • Validated encryption and monitoring configuration
