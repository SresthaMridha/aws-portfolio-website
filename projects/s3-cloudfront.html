Secure Static Website Deployment using S3 + CloudFront (OAC)

Deployed a production-style static website using Amazon S3 as a private origin and CloudFront as a secure CDN layer with HTTPS enforcement and Origin Access Control (OAC). Direct S3 access was intentionally blocked to ensure secure architecture.

Objective

  • Host a static website securely
  • Prevent direct S3 public access
  • Use CloudFront as CDN layer
  • Enforce HTTPS
  • Implement Origin Access Control (OAC)

Architecture Overview

User → CloudFront (HTTPS) → Secure Origin Request (OAC) → Private S3 Bucket

  • CloudFront handles public traffic
  • S3 bucket remains private
  • Direct S3 URL returns Access Denied
  • Edge caching enabled

S3 Bucket Configuration

  • Bucket Region: eu-west-2 (London)
  • Block Public Access: Enabled
  • Bucket Versioning: Enabled
  • MFA Delete: Disabled
  • Lifecycle rule: Non-current version expiration

CloudFront Configuration

  • Origin: Private S3 bucket
  • Origin Access Control (OAC) enabled
  • Viewer protocol policy: Redirect HTTP → HTTPS
  • Price class: All edge locations
  • Default root object: index.html

Security Validation

  • Verified CloudFront domain loads website successfully
  • Confirmed HTTPS enforced
  • Accessing S3 URL directly returns "Access Denied"
  • Public bucket access completely blocked

Outcome

Successfully implemented a secure static hosting architecture where CloudFront acts as the only public entry point, while the S3 bucket remains fully private. The configuration ensures security, performance, and controlled public exposure.
