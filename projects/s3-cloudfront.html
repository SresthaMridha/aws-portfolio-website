Secure Static Website Deployment with Amazon S3 and CloudFront

Architecture S3 + CloudFront CDN
Security Origin Access Control
Access Public S3 Blocked
Delivery HTTPS via CloudFront

This project demonstrates a secure static website architecture using Amazon S3 as the storage layer and Amazon CloudFront as the content delivery network (CDN). The S3 bucket remains private while CloudFront serves the website globally through edge locations using Origin Access Control (OAC).

Objective

  • Deploy a static website using AWS infrastructure
  • Prevent direct public access to the S3 bucket
  • Deliver website content securely via CloudFront CDN
  • Enable object versioning for recovery and change tracking

Architecture Overview

The architecture separates storage and delivery layers:

User → CloudFront CDN → Private S3 Bucket (Origin)

CloudFront handles HTTPS delivery, caching, and secure origin communication while the S3 bucket remains private with public access fully blocked.

S3 Storage Layer

  • Bucket region: eu-west-2 (London)
  • Bucket versioning enabled
  • Block Public Access: Enabled
  • Bucket used strictly as private origin storage

Public access is disabled to ensure objects cannot be accessed directly from S3.

Security Design

  • S3 Block Public Access enabled
  • CloudFront Origin Access Control (OAC) configured
  • Direct S3 URL access blocked
  • All content served only through CloudFront

This prevents origin bypass attacks where users attempt to access the storage layer directly.

CloudFront Distribution

  • CloudFront used as CDN layer
  • HTTPS enabled for secure delivery
  • Edge caching enabled
  • Origin Access Control configured for private S3 access

CloudFront retrieves content from the private S3 origin and distributes it globally through AWS edge locations.

Access Validation

  • Website successfully served via CloudFront distribution
  • Direct S3 access verified as blocked
  • AccessDenied error confirmed when bypassing CDN

This confirms that the S3 bucket is secured and accessible only through the CloudFront distribution.

Outcome

The deployment successfully implemented a secure static website architecture using Amazon S3 and CloudFront. By blocking direct S3 access and routing requests through CloudFront, the system ensures controlled origin access while benefiting from global edge caching and HTTPS delivery.

This architecture pattern is widely used for hosting static applications, documentation sites, and front-end web applications in production environments.
